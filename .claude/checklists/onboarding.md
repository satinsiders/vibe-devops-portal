# Developer Onboarding Checklist

Setup checklist for new team members.

---

## Day 1: Access & Setup

### Accounts
- [ ] GitHub account created/added to org
- [ ] Added to relevant repositories
- [ ] Slack/communication tool access
- [ ] Project management tool access (Jira, Linear, etc.)
- [ ] Documentation platform access (Notion, Confluence)
- [ ] Cloud provider access (if needed)

### Development Environment
- [ ] Git installed and configured
- [ ] Node.js installed (version: ______)
- [ ] Package manager installed (npm/pnpm/yarn)
- [ ] Code editor installed (VS Code recommended)
- [ ] Required VS Code extensions installed
- [ ] Docker installed (if used)

### Repository Setup
- [ ] Repositories cloned
- [ ] Dependencies installed (`npm install`)
- [ ] Environment variables configured (`.env.local`)
- [ ] Application runs locally
- [ ] Tests pass locally

---

## Day 2-3: Codebase Orientation

### Documentation
- [ ] Read CLAUDE.md (project guidelines)
- [ ] Read README.md
- [ ] Review architecture documentation
- [ ] Review API documentation
- [ ] Understand deployment process

### Codebase Exploration
- [ ] Understand project structure
- [ ] Review key directories:
  - [ ] `src/components/` - UI components
  - [ ] `src/services/` - Business logic
  - [ ] `src/api/` - API routes
  - [ ] `src/lib/` - Utilities
- [ ] Identify entry points
- [ ] Understand data flow

### Key Concepts
- [ ] Understand authentication flow
- [ ] Understand state management approach
- [ ] Understand database schema
- [ ] Understand API patterns
- [ ] Understand error handling

---

## Week 1: First Contribution

### Starter Tasks
- [ ] Complete first "good first issue"
- [ ] Make first pull request
- [ ] Receive code review feedback
- [ ] Merge first PR
- [ ] Deploy to staging (with guidance)

### Processes
- [ ] Understand git workflow (branching strategy)
- [ ] Understand PR review process
- [ ] Understand deployment process
- [ ] Understand on-call process (if applicable)

### Tools & Conventions
- [ ] Understand commit message format
- [ ] Understand code style guidelines
- [ ] Know how to run tests
- [ ] Know how to run linting
- [ ] Know how to debug locally

---

## Week 2-4: Independence

### Deeper Knowledge
- [ ] Understand CI/CD pipeline
- [ ] Understand monitoring/alerting
- [ ] Understand logging approach
- [ ] Know how to investigate issues
- [ ] Understand performance considerations

### Team Integration
- [ ] Attend all relevant meetings
- [ ] Participate in code reviews
- [ ] Pair programming session completed
- [ ] Know who to ask for different topics
- [ ] Comfortable asking questions

### Independent Work
- [ ] Complete medium-complexity task independently
- [ ] Handle a bug fix end-to-end
- [ ] Participate in on-call (shadow first)

---

## Claude Code Setup

### Installation
- [ ] Claude Code CLI installed
- [ ] Authenticated with Anthropic
- [ ] Permissions configured

### Configuration
- [ ] Review `.claude/rules/` files
- [ ] Review `.claude/skills/` files
- [ ] Review `.claude/commands/` files
- [ ] Understand available agents
- [ ] Practice using commands:
  - [ ] `/plan` - Planning
  - [ ] `/tdd` - Test-driven development
  - [ ] `/review-changes` - Code review
  - [ ] `/commit-push-pr` - Git workflow
